# Pokedex Management System (C)

## Overview

The **Pokedex Management System** is a command-line application written in C that manages Pokemon collections using fundamental data structures and algorithms. This project demonstrates proficiency in low-level programming, memory management, and algorithmic problem-solving, making it well-suited for applied mathematics and technical internship applications.

## Note on Design

This project is not a traditional Pokedex. Instead, it combines elements of both a Pokedex and a storage system.

* Each Pokemon is uniquely identified by its ID, and duplicates are not allowed.
* Only one instance of a Pokemon can exist per owner.
* When a Pokemon evolves, its previous form is removed rather than retained.

This approach prioritizes data integrity and efficient collection management over historical tracking.

## Features

* Binary Search Tree (BST) for efficient Pokemon storage and retrieval
* Breadth-First Search (BFS) and Depth-First Search (DFS) traversals
* Circular doubly linked list for managing multiple owners
* Queue implementation for level-order traversal
* Alphabetical sorting using qsort
* Pokemon evolution and battle simulation
* Pokedex merging functionality
* Robust dynamic memory management
* Safe and validated user input
* Modular design using header and source files
